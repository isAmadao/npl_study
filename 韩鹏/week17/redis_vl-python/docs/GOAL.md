# 开发最终目标文档

## 项目名称

**简易版 RedisVL** — 基于 Redis 和 Python 的轻量级 AI 缓存与检索系统

## 项目概述

实现一个最小化的 RedisVL 功能子集，包含四个核心模块：SemanticCache（语义缓存）、EmbeddingsCache（嵌入缓存）、SemanticMessageHistory（语义对话历史）、SemanticRouter（语义路由），并提供 Web 界面进行功能演示与交互。

---

## 最终交付物

### 后端

一个基于 FastAPI 的 Python Web 服务，集成以下四个功能模块：

| 模块 | 核心能力 | 触发条件 |
|------|---------|---------|
| **EmbeddingsCache** | 对文本嵌入向量做精确 hash 缓存，避免重复调用嵌入模型 | 每次文本向量化时自动检查 |
| **SemanticCache** | 对 LLM 的 prompt-response 做语义相似度缓存，语义相近的问题直接返回缓存结果 | 每次收到用户问题时自动检查 |
| **SemanticMessageHistory** | 以 session 维度存储对话历史，支持按语义检索相关历史消息 | 每次对话时存储，查询时按需检索 |
| **SemanticRouter** | 根据预定义路由规则，将用户问题按语义匹配到对应处理分支 | 每次收到用户问题时自动匹配 |

### 前端

一个简洁的 Web 页面，包含：

- **输入区域**：文本输入框 + 发送按钮 + Session ID 选择
- **结果展示区域**：四个卡片/面板，分别展示上述四个功能模块的调用结果
  - EmbeddingsCache：命中/未命中状态 + 向量摘要
  - SemanticCache：命中/未命中状态 + 缓存内容
  - SemanticMessageHistory：本轮存储确认 + 检索到的相关历史
  - SemanticRouter：匹配到的路由名称 + 匹配距离

### 基础设施

- Redis 服务：通过 Docker 容器运行 Redis Stack
- Python 后端：FastAPI + uvicorn，运行在 conda 环境 `py310`
- 嵌入模型：优先使用 `C:\Users\peng\.cache\huggingface\hub` 下已缓存的 HuggingFace 模型

## 技术栈选型

| 层次 | 技术 | 说明 |
|------|------|------|
| 后端框架 | FastAPI | 高性能异步 Web 框架 |
| Redis 客户端 | redis-py | Redis 官方 Python 客户端 |
| 向量嵌入 | sentence-transformers | 使用本地 HuggingFace 模型 |
| Web 前端 | 原生 HTML + Vanilla JS | 单页面，无需前端框架 |
| 容器化 | Docker (Redis Stack) | 通过 docker-compose 管理 |
| 测试 | pytest + httpx | 自动化测试 |
| Python 环境 | conda (py310) | 用户指定 |

## 非功能需求

- 代码简洁，每个模块核心代码控制在 200 行以内
- 配置集中管理（环境变量或 .env 文件）
- Redis 连接失败时有明确的错误提示
- 项目结构清晰，模块间低耦合
- 自动化测试覆盖每个版本的新增功能

## 不包含的内容（明确排除）

- 不做用户认证/授权
- 不做分布式部署
- 不做性能压测
- 不做生产级日志/监控
- 不做前端路由（SPA）
- 不做数据库持久化（仅使用 Redis）
- 不做异步向量化（简化实现，使用同步方式）
