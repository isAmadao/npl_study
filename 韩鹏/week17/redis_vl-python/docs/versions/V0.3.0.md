# V0.3.0 — SemanticCache 模块

## 版本目标

实现 SemanticCache 模块：对 prompt-response 做语义相似度缓存，语义相似的问题直接返回缓存结果。更新前端展示卡片，显示缓存命中/未命中状态、匹配距离和缓存内容。

## 前置条件

- V0.2.0 已完成（EmbeddingsCache 可用）
- 向量化器可用
- Redis Stack 的 RediSearch 模块可用（redis-stack 镜像自带）

## 开发任务清单

### 任务 1：实现 SemanticCache 模块

- [x] 创建 `app/modules/semantic_cache.py`
  - 类 `SemanticCache`
  - `__init__(self, redis_client, vectorizer, name="semcache", distance_threshold=0.2)`
  - `store(prompt, response, metadata=None)` — 存储 prompt+response+向量
  - `check(prompt)` — 语义搜索缓存
    - 向量化 prompt
    - 用 RediSearch FT.SEARCH 执行 KNN 搜索
    - 如果有匹配（距离 < threshold），返回缓存结果
    - 无匹配返回 None

**实现要点：**

- 存储结构：每个缓存项存储为 Redis Hash
  - key 格式：`semcache:{name}:{uuid}`
  - 字段：prompt, response, prompt_vector（binary float32 bytes，用于向量索引）, prompt_vector_hex（hex 备份）, created_at
- 向量索引：
  - 用 `FT.CREATE` 创建索引
  - 字段：prompt_vector (VECTOR, FLAT, 6 params), prompt (TEXT), response (TEXT), created_at (NUMERIC)
  - 距离度量：COSINE
- 搜索：`FT.SEARCH idx "*=>[KNN $K @prompt_vector $vec AS dist]" SORTBY dist LIMIT 0 1 PARAMS 2 vec <binary> DIALECT 2`
- 距离阈值：COSINE 距离 ≤ distance_threshold 视为命中

## 技术细节：RediSearch 向量搜索

```python
# 创建索引（仅首次）
FT.CREATE idx:semcache:{name}
  ON HASH PREFIX 1 semcache:{name}:
  SCHEMA
    prompt TEXT
    response TEXT
    prompt_vector VECTOR FLAT 6
      TYPE FLOAT32
      DIM {dim}
      DISTANCE_METRIC COSINE
    created_at NUMERIC SORTABLE

# KNN 搜索
FT.SEARCH idx:semcache:{name}
  "*=>[KNN 1 @prompt_vector $vec AS dist]"
  SORTBY dist
  LIMIT 0 1
  PARAMS 2 vec {binary_vector}
  DIALECT 2
```
