# V0.5.0 — SemanticRouter 模块

## 版本目标

实现 SemanticRouter 模块：根据预定义路由规则，将用户问题按语义匹配到对应处理分支。同步更新前端展示卡片，显示匹配到的路由名称、匹配距离和参考短语。

## 前置条件

- V0.4.0 已完成，SemanticMessageHistory 可用
- RediSearch 向量检索能力已验证

### 任务 2：实现 SemanticRouter 模块

- [x] 创建 `app/modules/semantic_router.py`
  - 类 `SemanticRouter`
  - `__init__(self, redis_client, vectorizer, name="router")`
  - `add_routes(routes)`：批量添加路由及参考短语
  - `route(query)`：路由查询并返回匹配结果
  - `has_routes()`：判断当前 router 是否已有路由数据

**实现要点：**

- 每个参考短语用 Redis Hash 存储
- key 格式：`router:{name}:{route_name}:{ref_index}`
- Hash 字段：`route_name`、`reference`、`vector`、`threshold`
- RediSearch 索引字段包含 `route_name`（TAG）、`reference`（TEXT）、`vector`（VECTOR）、`threshold`（NUMERIC）
- 先做 KNN 搜索，再判断距离是否小于等于该路由自己的 threshold

## 手动测试流程

下面流程用于本地人工验证 V0.5.0 的完整效果。

1. 启动 Redis Stack。
   - 推荐使用 `docker compose up -d`
   - 确认 Redis 监听在 `6379`
2. 激活 Python 环境。
   - 推荐：`conda activate py310`
3. 启动 Web 服务。
   - 执行：`python run.py`
   - 终端应显示 Redis 可连通、FastAPI 已启动
4. 打开浏览器访问页面。
   - 地址：`http://localhost:8000`
5. 输入第一轮问题与 Session ID。
   - `Session ID` 示例：`demo-session-a`
   - 问题示例：`Python 列表怎么排序`
6. 点击“发送”或按 Enter 提交。
   - 观察 4 个卡片的返回结果
   - `EmbeddingsCache` 首次应显示未命中，第二次相同输入应命中
   - `SemanticCache` 首次应显示未命中，第二次相同输入应命中
   - `SemanticMessageHistory` 应显示“✅ 本轮已存储”
   - `SemanticRouter` 应匹配到“技术问答”
7. 再次发送相同问题。
   - 检查 `EmbeddingsCache` 变为命中
   - 检查 `SemanticCache` 变为命中
   - 检查 `MessageHistory` 的相关历史条数增加
   - 检查 `SemanticRouter` 仍能稳定命中同一路由
8. 切换到另一个 Session ID。
   - 例如改为 `demo-session-b`
   - 再次输入相同问题
   - 检查 `MessageHistory` 中没有串会话的数据
9. 输入一条明显不相关的问题。
   - 示例：`给我推荐一个投资策略`
   - 检查 `SemanticRouter` 显示“未匹配到路由”
10. 结合浏览器结果确认页面布局。
    - 四张卡片都能正确展示
    - 加载状态和命中/未命中颜色切换正常
