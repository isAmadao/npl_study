# V0.1.0 — 项目脚手架与基础设施

## 版本目标

搭建项目基础框架：启动 Redis Stack 容器，搭建 FastAPI 后端骨架，配置 Python 环境与依赖，创建前端页面框架（无功能仅布局），确保所有基础设施可运行并通过冒烟测试。

## 前置条件

- Windows 11 系统
- conda 环境 `py310` 已就绪
- Docker Desktop 已安装并运行

## 验收标准

- [x] `docker-compose up -d` 可成功启动 Redis Stack（在本机端口保留限制下，通过 `.env` 配置 `REDIS_HOST_PORT=16379`、`REDIS_INSIGHT_PORT=18001`）
- [x] `pip install -r requirements.txt` 无报错
- [x] `python run.py` 可启动 Web 服务（Redis 与应用端口支持环境变量覆盖，已在本机通过 `.env` 的 `REDIS_PORT=16379`、`APP_PORT=18000` 验证）
- [x] 浏览器访问 `http://localhost:<APP_PORT>` 可看到页面布局（本机验证端口：`18000`）
- [x] `GET /api/health` 返回 `{"status": "ok"}`
- [x] `pytest tests/ -v` 全部通过

> 说明：当前 Windows 环境存在端口排除范围（含 6379、8001、8000），因此保留默认端口配置同时新增了宿主端口覆盖能力，确保文档默认值和本机可运行性兼容。
