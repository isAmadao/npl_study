# V0.4.0 — SemanticMessageHistory 模块

## 版本目标

实现 SemanticMessageHistory 模块：按 session 维度存储对话历史，并支持按语义检索相关历史消息。同步更新前端展示卡片，显示本轮存储确认和检索到的历史消息摘要。

## 前置条件

- V0.3.0 已完成，SemanticCache 可用
- Redis Stack 的 RediSearch 向量检索能力已验证

## 开发任务清单

### 任务 1：实现 SemanticMessageHistory 模块

- [x] 创建 `app/modules/message_history.py`
  - 类 `SemanticMessageHistory`
  - `__init__(self, redis_client, vectorizer, name="msghist", session_tag="default", distance_threshold=0.5)`
  - `store(prompt, response)`：存储一轮对话
  - `get_recent(top_k=5)`：按时间获取最近 N 轮对话
  - `get_relevant(query, top_k=5)`：按语义检索相关历史消息
  - `clear()`：清除当前 session 的所有历史

**实现要点：**

- key 格式：`msghist:{name}:{session_tag}:{uuid}`
- session 隔离通过 key prefix 实现：`msghist:{name}:{session_tag}:`
- 存储时自动向量化 prompt
- RediSearch 索引字段包含 `prompt`、`response`、`prompt_vector`、`created_at`

### 任务 2：更新 FastAPI

- [x] 更新 `app/main.py` 中的 `POST /api/chat`

  - 返回 `message_history` 字段，包含：
