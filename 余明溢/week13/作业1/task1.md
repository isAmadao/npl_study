# DeepSeek-V3 相比 DeepSeek-V2 的主要改进

## 一、模型架构改进

### 1. 无辅助损失负载均衡策略
- **V2**：依赖辅助损失来鼓励专家负载均衡，可能损害模型性能  
- **V3**：引入 **auxiliary-loss-free** 负载均衡策略，通过动态调整每个专家的偏置项来控制路由，无需辅助损失，同时实现更优的负载均衡和模型性能

### 2. 多 Token 预测目标
- **V2**：仅预测下一个 token  
- **V3**：引入 **Multi-Token Prediction (MTP)** 目标，每个位置预测多个未来 token，提升训练效率和模型性能，并可用于推理阶段加速解码（如 speculative decoding）

### 3. 专家选择机制优化
- **V3** 使用 sigmoid 函数计算 token 与专家的亲和度，并对选中的亲和度分数进行归一化，与 V2 略有不同，提升了路由稳定性

---

## 二、训练效率与基础设施优化

| 优化项 | DeepSeek-V2 | DeepSeek-V3 |
|--------|-------------|-------------|
| **精度格式** | BF16 | **FP8 混合精度训练**（首次在 671B 模型上成功应用） |
| **流水线并行** | 传统方法 | **DualPipe 算法**，实现计算与通信完全重叠，减少流水线气泡 |
| **跨节点通信** | 基础 All-to-All | 限制 token 到 4 节点 + NVLink/IB 重叠 + warp specialization |
| **显存优化** | 常规 | 重计算 RMSNorm 和 MLA up-projection；EMA 参数存 CPU；MTP 模块共享 embedding/输出头 |

## 三、数据与训练规模对比

| 项目 | DeepSeek-V2 | DeepSeek-V3 |
|------|-------------|-------------|
| 总参数量 | 236B | **671B** |
| 激活参数量 | 21B | **37B** |
| 训练 token 数 | 约 8.1T | **14.8T** |
| 数学与代码数据 | 少量 | **显著增加** |
| FIM 策略 | 无 | **引入（Fill-in-Middle）** |
| 多语言支持 | 中英为主 | **扩展多语言** |

---

## 四、后训练改进

### 1. 从 DeepSeek-R1 蒸馏推理能力
- 利用 R1 模型生成高质量推理数据（数学、代码等），结合系统提示与 RL 微调，提升推理性能

### 2. 自奖励机制
- 在无外部反馈的场景中，使用 V3 自身投票结果作为奖励信号，结合 Constitutional AI 进行对齐

### 3. MTP 用于解码加速
- V3 的 MTP 模块在推理阶段可被重用于 speculative decoding，实现 **1.8 倍解码速度提升**

---

## 五、部署优化

- **Prefilling 阶段**：4 节点 32 GPU 最小部署单元，支持 TP4 + EP32，引入冗余专家策略平衡负载  
- **Decoding 阶段**：40 节点 320 GPU，每个 GPU 只部署一个专家，结合 IBGDA 技术降低延迟

## 七、训练成本与稳定性

- **总训练成本**：2.788M H800 GPU 小时（约 557 万美元），相比 V2 显著降低每 token 成本  
- **训练稳定性**：全过程中无不可恢复的 loss spike，无需回滚
