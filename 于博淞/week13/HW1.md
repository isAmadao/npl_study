# DeepSeek-V3 相比 V2 的核心改进

V3 沿用了 V2 的MLA 注意力和 DeepSeekMoE，主要改动集中在下面几点：

- **规模变大**：总参数从 236B 提到 671B，激活参数从 21B 提到 37B，训练数据从 8.1T 提到 14.8T tokens。
- **负载均衡换思路**：抛掉 V2 那套靠 auxiliary loss 强推均衡的做法，改用每个专家加一个动态偏置项的"无辅助损失"策略，过载就调低、欠载就调高。专家分布更均衡，性能也没被均衡这件事拖累。
- **多 token 预测 (MTP)**：训练时不只预测下一个 token，还顺带预测后面 1 个，训练信号更密。推理时这部分还能直接拿来做 speculative decoding，吐字速度大概翻 1.8 倍。
- **FP8 混合精度训练**：V2 是 BF16，V3 把大部分 GEMM 换成 FP8，配合细粒度量化（激活按 1×128、权重按 128×128 分块）和 FP32 累加补偿精度。这是首次在这种规模上把 FP8 训练跑通。
- **训练框架重写**：自研 DualPipe 流水线把通信和计算重叠掉，加上手写的跨节点 all-to-all 通信 kernel，2048 张 H800 上 14.8T tokens 只花了 2.788M GPU 小时（约 558 万美元）。
- **后训练蒸馏 R1**：把 DeepSeek-R1 的长链推理能力蒸馏进 V3，数学和代码 benchmark 涨幅明显，但 V3 输出还是保持简洁不啰嗦。
