## MinerU 论文要点

- 论文：[MinerU: An Open-Source Solution for Precise Document Content Extraction](https://arxiv.org/abs/2409.18839)（上海 AI 实验室，2024）
- 目标：高精度 PDF 内容抽取，服务 LLM 训练、RAG、Agent
- 路线：**多模块解析**——版面分析 → 分区域识别（文本 OCR、公式→LaTeX、表格→HTML、图片导出）
- 流程：预处理（PyMuPDF）→ PDF-Extract-Kit 解析 → 后处理（去页眉页脚、阅读顺序）→ 输出 Markdown/JSON
- 优势：适应复杂版式、扫版 PDF、多栏文档；输出可直接用于 RAG

## MinerU 使用方式

- **命令行**：`mineru -p <PDF> -o <输出目录>`
- **API 服务**：`mineru-api --host 0.0.0.0 --port 8000`
- **在线工具**：https://mineru.net/OpenSourceTools/Extractor（零安装试解析）
- **文档**：https://opendatalab.github.io/MinerU/usage/quick_usage/

## pdfplumber 特点

- 基于 pdfminer，按页抽取**文本和坐标**
- 轻量、CPU 即可、速度快
- 适合有文字层的简单 PDF
- 不支持版面理解、公式 LaTeX、扫版 OCR、图注语义

## 效果差异

| 对比项 | pdfplumber | MinerU |
|--------|------------|--------|
| 输出 | 纯文本为主 | 结构化 Markdown + 图片路径 |
| 复杂版式/多栏 | 易乱序 | 按阅读顺序 |
| 公式/表格 | 差或乱码 | LaTeX / HTML |
| 扫描件 | 常无文本 | 自动 OCR |
| 速度/资源 | 快、无 GPU | 慢，建议 GPU/离线队列 |
| RAG 场景 | 需大量清洗 | 更适合多模态 RAG |
