# 作业1：多模态RAG聊天机器人 FastAPI 实现

## 需求文档

### 1. 项目概述

将 05-multimodal-rag-chatbot-my 项目从 Streamlit 单页应用转换为模块化的 FastAPI 服务，支持：
- 文档上传与解析（异步 Kafka 队列）
- 多模态检索（文本 + 图像）
- 多模态问答（RAG）

### 2. 技术架构

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   FastAPI   │────▶│    Kafka    │────▶│   Worker    │
│   Server    │     │    Queue    │     │  (Consumer) │
└─────────────┘     └─────────────┘     └─────────────┘
      │                                        │
      │                                        ▼
      │                               ┌─────────────┐
      │                               │   MinerU    │
      │                               │  (Parser)   │
      │                               └─────────────┘
      │                                        │
      ▼                                        ▼
┌─────────────┐                       ┌─────────────┐
│    SQLite    │                       │   Milvus    │
│  (Metadata)  │                       │  (Vectors)  │
└─────────────┘                       └─────────────┘
```

### 3. 接口定义

#### 3.1 文档上传接口
```
POST /api/v1/documents/upload
Content-Type: multipart/form-data

Request:
  - file: UploadFile (pdf, docx, txt)
  - knowledge_base_id: int (optional, default=1)

Response:
  {
    "code": 0,
    "message": "success",
    "data": {
      "document_id": int,
      "filename": str,
      "file_path": str,
      "status": "pending"
    }
  }
```

#### 3.2 文档解析状态查询
```
GET /api/v1/documents/{document_id}/status

Response:
  {
    "code": 0,
    "message": "success",
    "data": {
      "document_id": int,
      "filename": str,
      "status": "pending|processing|completed|failed",
      "chunks_count": int
    }
  }
```

#### 3.3 多模态问答接口
```
POST /api/v1/chat

Request:
  {
    "question": str,
    "knowledge_base_id": int (optional),
    "top_k": int (default=5)
  }

Response:
  {
    "code": 0,
    "message": "success",
    "data": {
      "answer": str,
      "sources": [
        {
          "text": str,
          "file_name": str,
          "file_path": str,
          "score": float
        }
      ]
    }
  }
```

#### 3.4 文档列表接口
```
GET /api/v1/documents

Response:
  {
    "code": 0,
    "message": "success",
    "data": [
      {
        "id": int,
        "filename": str,
        "filepath": str,
        "filestate": str,
        "created_at": datetime
      }
    ]
  }
```

#### 3.5 文档删除接口
```
DELETE /api/v1/documents/{document_id}

Response:
  {
    "code": 0,
    "message": "success"
  }
```

### 4. 数据模型

#### 4.1 SQLite (files 表)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer | 主键 |
| filename | String(255) | 原始文件名 |
| filepath | String(1000) | 存储路径 |
| filestate | String(20) | 处理状态 |
| created_at | DateTime | 创建时间 |

#### 4.2 Milvus Collection (rag_data_new)
| 字段 | 类型 | 说明 |
|------|------|------|
| text_vector | FloatVector(512) | BGE 文本向量 |
| clip_text_vector | FloatVector(1024) | CLIP 文本向量 |
| clip_image_vector | FloatVector(1024) | CLIP 图像向量 |
| text | VarChar(4000) | 文本内容 |
| db_id | Int | 文档 ID |
| file_name | VarChar(255) | 文件名 |
| file_path | VarChar(1000) | 文件路径 |

### 5. Worker 流程

```
1. Kafka Consumer 消费 "rag-data" topic
2. 获取文件信息 (file_name, file_path, id)
3. 调用 MinerU 解析 PDF → Markdown + images
4. 文本分块 (chunk_size=256)
5. BGE 编码文本向量
6. CLIP 编码文本/图像向量
7. 批量插入 Milvus
8. 更新 SQLite 状态
```

### 6. 依赖组件

- FastAPI + Uvicorn
- SQLAlchemy 2.0 (async)
- pymilvus
- kafka-python
- sentence-transformers (BGE, CLIP)
- aiofiles (异步文件处理)
