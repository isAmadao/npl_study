# SemanticRouter 文档

## 概述

语义路由模块，基于 Embedding 相似度将用户查询分发到不同的处理流程。

## 路由策略

### 中心向量匹配
1. 每个路由注册多个示例查询
2. 示例查询的 Embedding 均值作为路由的「中心向量」
3. 新查询与各路由中心向量计算相似度
4. 最高相似度 > threshold → 路由到该意图

### 兜底机制
- 所有路由的相似度均低于阈值时，走 `fallback` 兜底路由
- 兜底路由名称可通过 `default_route` 参数配置

### 动态热加载
支持从 JSON 配置文件热加载路由定义：

```json
{
  "routes": [
    {
      "name": "greeting",
      "description": "问候类查询",
      "examples": ["你好", "hello", "早上好"]
    },
    {
      "name": "weather",
      "description": "天气查询",
      "examples": ["今天天气", "下雨了吗"]
    }
  ]
}
```

## 使用场景

1. **多 Agent 路由**：将查询分发到不同专业 Agent
2. **意图分类**：区分知识问答、代码生成、数据分析等
3. **工具选择**：选择调用哪个工具或 API
4. **对话策略**：根据意图切换对话策略

## 最佳实践

- 每个路由至少提供 5~10 个示例查询
- 示例应覆盖该路由的典型表达方式
- 路由名称应具有明确的语义
- 置信度阈值建议从 0.7 开始调试
