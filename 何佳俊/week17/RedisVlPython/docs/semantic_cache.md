# SemanticCache 文档

## 概述

语义缓存模块，通过 Embedding 相似度匹配避免重复的 LLM 调用。

## 核心流程

```
用户查询 ──→ 生成 Embedding ──→ 向量相似度搜索
                                    │
                     ┌──────────────┴──────────────┐
                     │                              │
              相似度 > 阈值                    相似度 < 阈值
                     │                              │
                     ▼                              ▼
              返回缓存回答 ──── 调用 LLM 生成回答
                                      │
                                      ▼
                              存储 Q&A + Embedding
```

## 阈值调优指南

| 阈值范围 | 效果 | 适用场景 |
|---------|------|---------|
| 0.95~1.0 | 几乎只命中完全相同的查询 | 精确匹配场景 |
| 0.85~0.95 | 命中高度相似的查询 | 通用推荐 |
| 0.70~0.85 | 命中语义相似的查询 | 意图模糊场景 |
| 0.50~0.70 | 大量命中 | 低精度可接受场景 |

## 淘汰策略

### LRU (Least Recently Used)
默认策略。淘汰最久未被访问的条目。

### LFU (Least Frequently Used)
淘汰访问频率最低的条目。适合访问模式不均匀的场景。

### TTL
基于时间的过期淘汰。所有条目具有固定的生存时间。

## 批量操作

支持批量存储和读取，减少网络往返：

```python
# 批量存储
cache.batch_set([
    ("key1", vec1, meta1),
    ("key2", vec2, meta2),
])

# 批量读取
vectors = cache.batch_get(["key1", "key2"])
```
