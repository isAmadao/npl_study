## 项目概述

**evo-werewolf-agent** — 基于多 Agent 协作框架的狼人杀自主博弈系统。每个 Agent 根据其扮演角色（狼人、预言家、女巫等）拥有独立的目标、策略与行动空间，在严格信息隔离的约束下进行推理、发言与决策。

## 当前状态

后端核心引擎已完成，模拟模式可运行完整对局。前端具备完整人类玩家交互（发言/投票/夜晚行动），已通过压力测试。

**已完成：**

## 环境配置

本项目使用 conda 虚拟环境 `evo-werewolf`（Python 3.12）作为基础解释器，搭配 uv 管理依赖。

```bash
# 激活 conda 环境
conda activate evo-werewolf

# 安装后端依赖
cd backend
uv sync

# 运行模拟对局
cd backend && uv run python ../scripts/run_game.py

# 批量评测
cd backend && uv run python ../scripts/batch_eval.py --count 50
```

如果 `uv` 命令不可用，使用 conda 环境中的 uv：
```bash
cd backend
"$CONDA_PREFIX/Scripts/uv" run python ../scripts/run_game.py
```

> 注：conda 环境 `evo-werewolf` 已安装 uv，`.python-version` 已固定到该环境。在 backend 目录下运行 `uv run` 时自动使用 conda 环境的 Python。

## 常用命令

```bash
# 激活环境
conda activate evo-werewolf

# 后端 - 安装依赖
cd backend
uv sync

# 后端 - 启动服务
cd backend && uv run uvicorn src.werewolf.main:app --reload --port 8000

# 后端 - 运行模拟对局
cd backend && uv run python ../scripts/run_game.py              # 单局
cd backend && uv run python ../scripts/run_game.py --count 10   # 10局
cd backend && uv run python ../scripts/batch_eval.py --count 50 # 批量评测

# 后端 - 运行测试
cd backend && uv run pytest

# 后端 - 使用 LLM 模式（需设置 API key）
cd backend && uv run python ../scripts/run_game.py --llm

# 后端 - 启动服务（带热重载）
cd backend && uv run uvicorn src.werewolf.main:app --reload --port 8000

# 压力测试（需先启动后端服务）
cd backend && uv run python ../scripts/stress_test/engine_stress.py      # 引擎并发
cd backend && uv run python ../scripts/stress_test/endurance_test.py     # 耐力测试
# API/WS 压力测试需要后端运行中：
cd backend && uv run python ../scripts/stress_test/api_stress.py         # API 负载
cd backend && uv run python ../scripts/stress_test/ws_stress.py          # WS 压力

# 人类玩家交互诊断
cd backend && uv run python ../scripts/stress_test/diagnose_human.py

# 前端
cd frontend
pnpm install
pnpm dev      # Vite dev server (localhost:5173)
pnpm build
```

## 核心设计原则

1. **信息隔离是第一优先级**：Agent 绝对不能看到不该看到的信息，使用 PlayerView 按 Visibility 过滤事件日志
2. **事件日志是唯一真相来源**：所有状态变更必须通过 Event 记录，事件标注 visibility 字段控制可见范围
3. **结构化输出**：LLM 必须返回 Pydantic 模型格式数据，解析失败则重试或降级
4. **可观测性**：每一步 Agent 决策必须附带 reasoning trace
5. **状态机驱动**：游戏流程由 GamePhase 状态机控制，GameRunner 调度各阶段流转
