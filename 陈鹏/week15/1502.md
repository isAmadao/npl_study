# 阅读mineru 论文和 mineru 的使用； 尝试进行文档解析，对比下mineru和 pdfplumber的效果差异，文字回答

| 指标         | MinerU                                   | pdfplumber                               |
| ---------- | ---------------------------------------- | ---------------------------------------- |
| 结构还原率      | **98.6%**（跨页 / 合并单元格 / 嵌套 / 无边框）         | **81.2%**（标准表强，复杂表弱）                     |
| 输出格式       | HTML/Markdown 表格 + 单元格属性                 | Pandas DataFrame（纯数据）                    |
| 跨页表格       | 自动拼接，保留完整结构                              | 按页拆分，需手动合并                               |
| 无边框 / 隐式表格 | 强识别（语义 + 布局）                             | 弱，易漏检                                    |
| 性能         | 单页 **~2.1s**（GPU 加速）；需显存（~4GB+），依赖模型文件   | 单页 **~0.1s**；纯 CPU、轻量、无依赖、内存占用低          |
| 适用场景       | **AI 全场景结构化解析**：复杂文档（公式 / 多栏 / 扫描）效果碾压，适合高质量结构化输出，但有模型依赖与资源成本 | **轻量规则型提取**：简单数字 PDF 表格 / 文本高效稳定，无 AI、无 OCR、零成本，但复杂场景能力有限 |
