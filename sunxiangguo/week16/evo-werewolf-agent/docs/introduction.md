# evo-werewolf-agent 项目介绍

## 📖 项目概述

**evo-werewolf-agent** 是一个基于大语言模型（LLM）的**多智能体狼人杀游戏系统**，展示了AI智能体在复杂博弈场景中的协作与对抗能力。

### 核心目标

构建一个具备以下特性的狼人杀AI系统：
1. **高度自主决策** - 每个AI玩家都能基于有限信息独立思考和决策
2. **严格信息隔离** - 防止"开天眼"作弊，确保公平博弈
3. **完整游戏流程** - 支持标准狼人杀规则的所有阶段
4. **可观测性** - 全程记录AI的思考过程和决策逻辑
5. **经验进化** - Agent能从历史对局中学习并改进策略

### 项目亮点

✨ **多智能体架构**: 每个玩家都是独立的AI Agent，具备完整的决策能力

🔒 **信息隔离机制**: 通过私有上下文实现严格的信息权限控制

🧠 **记忆与学习**: 跨游戏经验存储，Agent能"记住"并应用历史教训

🎮 **双模式运行**: 支持命令行演示和Web界面观战

📊 **结构化日志**: 完整记录游戏过程，支持复盘分析

🔌 **RESTful API**: 提供完整的HTTP接口，便于集成和扩展

## 🎭 游戏规则

### 标准板子配置

#### 标准6人局
- **狼人阵营** (2人): 狼人 × 2
- **好人阵营** (4人): 预言家、女巫、猎人、村民

#### 简易4人局
- **狼人阵营** (1人): 狼人 × 1
- **好人阵营** (3人): 预言家、女巫、村民

#### 标准9人局
- **狼人阵营** (3人): 狼人 × 3
- **好人阵营** (6人): 预言家、女巫、猎人、村民 × 2、白痴

### 胜负条件

采用**屠边规则**:
- **好人胜利**: 消灭所有狼人
- **狼人胜利**: 所有神职死亡 **或** 所有平民死亡

### 角色能力

| 角色 | 阵营 | 夜间能力 | 特殊规则 |
|------|------|----------|----------|
| 狼人 | 邪恶 | 集体投票杀人 | 不能自杀 |
| 预言家 | 善良 | 查验一人身份 | 仅返回好人/狼人 |
| 女巫 | 善良 | 解药救人 / 毒药杀人 | 双药互斥，不可自救 |
| 猎人 | 善良 | 无 | 死亡时开枪（被毒不能） |
| 村民 | 善良 | 无 | 纯靠推理 |

### 游戏流程

```
第1天开始
  ↓
夜晚阶段
  ├─ 狼人杀人 (集体投票)
  ├─ 预言家查验
  └─ 女巫用药 (解药/毒药)
  ↓
夜晚结果结算
  ↓
白天阶段
  ├─ 宣布昨晚死亡
  ├─ 公开演讲 (所有存活玩家)
  └─ 投票放逐
  ↓
检查胜负 → 游戏结束 / 进入下一天
```

## 🚀 快速开始

### 环境要求

- Python 3.10+
- Node.js 16+ (如需使用前端界面)
- 通义千问API密钥 (或其他OpenAI兼容API)

### 安装步骤

```bash
# 1. 克隆或进入项目目录
cd evo-werewolf-agent

# 2. 安装Python依赖
pip install -r requirements.txt

# 3. 配置API密钥
# 编辑 config/system_config.json
{
  "base_url": "你的API地址",
  "api_key": "你的API密钥",
  "default_model": "模型名称"
}
```

### 运行方式

#### 方式一：命令行演示（推荐新手）

```bash
# 默认6人局，自动执行
python main_demo.py

# 4人局
python main_demo.py -c simple_4

# 交互模式（每步询问）
python main_demo.py -i

# 逐天模式（每天询问）
python main_demo.py -d

# 自定义决策风格
python main_demo.py -s '{"0":"bold", "1":"cautious"}'
```

#### 方式二：API服务器 + 前端界面

**启动后端**:
```bash
uvicorn api.server:app --reload --host 0.0.0.0 --port 8000
```

访问 API 文档: http://localhost:8000/docs

**启动前端** (新终端窗口):
```bash
cd frontend
npm install
npm run dev
```

访问前端界面: http://localhost:5173

#### 方式三：纯API调用

```bash
# 创建游戏
curl -X POST http://localhost:8000/games \
  -H 'Content-Type: application/json' \
  -d '{"config_name": "standard_6"}'

# 推进游戏 (替换 YOUR_GAME_ID)
curl -X POST http://localhost:8000/games/YOUR_GAME_ID/step

# 查看游戏状态
curl http://localhost:8000/games/YOUR_GAME_ID
```

## 💡 核心特性

### 1. 决策风格系统

可为不同玩家设置不同的决策风格：

- **bold** (大胆): 倾向于激进策略，如悍跳预言家、强势指控
- **cautious** (谨慎): 倾向于保守策略，如低调隐藏、谨慎投票
- **balanced** (平衡): 综合权衡，灵活应对
- **random** (随机): 随机选择风格，增加不确定性

### 2. 经验记忆系统

Agent能够：
- 📝 游戏结束后自动保存经验教训
- 🔍 新游戏开始时检索相关历史经验
- 🧠 将经验融入决策Prompt，改进行为

经验文件存储在 `memory/experiences/` 目录，按角色分类。

### 3. 结构化日志

每局游戏生成两类日志：

**游戏记录** (`logs/game_*.json`):
- 完整的游戏状态快照
- 角色分配、对话历史、死亡记录
- 可用于复盘分析

**运行日志** (`logs/game_*_run_log.json`):
- 结构化事件日志
- 分级记录（DEBUG/INFO/WARNING/ERROR）
- 记录类型：行动、发言、投票、死亡

### 4. 游戏控制器

支持三种控制模式：

- **自动模式** (`MODE_AUTO`): 全自动执行，无需干预
- **逐天模式** (`MODE_DAY_BY_DAY`): 每天开始前等待确认
- **交互模式** (`MODE_INTERACTIVE`): 每阶段都等待用户输入

### 架构分层

```
┌─────────────────────────────────────┐
│      前端展示层 (Vue 3)             │
```

### 关键技术

- **后端**: FastAPI + Uvicorn
- **前端**: Vue 3 + Vite
- **AI框架**: openai-agents SDK
- **LLM**: 通义千问 (可替换其他OpenAI兼容模型)
- **数据验证**: Pydantic v2
- **测试**: pytest + pytest-asyncio

## 🔮 未来发展方向

### 方向一：自适应进化系统

实现 Agent 的自我进化：
```
评估当前策略 → 修改Prompt → 重新测试 → 持续优化
```

让Agent在没有人工干预的情况下，自我演化出适合各角色的专业策略。

### 方向二：评测与天梯榜

建立可量化的评价体系：
- 📈 发言煽动性评分
- 🧩 逻辑严密性评分
- 🎭 挡刀成功率
- 🏆 不同模型的胜率排行榜

由"AI复盘官"进行归因分析，例如：
> "狼人赢了是因为3号玩家聊爆了，而不是4号玩家悍跳成功"

### 方向三：闭环自进化 (RLAIF)

实现完整的强化学习循环：
```
对局 → 数据回流 → 归因分析 → 策略微调 → 再对局
```

让Agent在连续玩了100局后，自发学会高阶战术：
- 倒钩狼（假装好人阵营）
- 垫飞（故意误导投票）
- 滴滴代跳（暗示队友跳身份）
